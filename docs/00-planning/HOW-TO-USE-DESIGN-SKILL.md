# 如何使用产品设计协作功能

> 本文档说明如何与 AI 助手进行产品设计头脑风暴

---

## 一、快速开始

### 方式1：直接描述你的想法

只需直接说出你的产品设计想法，AI 会自动识别并启动头脑风暴模式：

```
"在产品端，我建议改用硅胶材料，防水好打理"
```

AI 会自动：
1. 读取相关背景文档
2. 分析当前方案
3. 提供多个改进方案
4. 与你讨论确认
5. 更新项目文档

### 方式2：使用关键词触发

在描述中使用以下关键词，会自动触发相应的设计讨论模式：

| 关键词 | 触发模式 | AI 会读取的文档 |
|--------|---------|----------------|
| 硬件/外壳/材料/设计 | 硬件设计讨论 | hardware-specification.md, character-ip-design.md |
| 功能/新功能/增加功能 | 功能设计讨论 | mvp-specification.md, api-specification.md |
| 定价/会员/收费 | 商业模式讨论 | pricing-strategy.md |
| APP/界面/UI | APP设计讨论 | app-ux-design.md, ui-design-system.md |

## 二、典型使用场景

### 场景1：改进现有设计

**你说：**
```
我觉得毛绒外壳不好清洗，建议用硅胶材料
```

**AI 会做：**
1. 读取当前硬件规格文档
2. 分析毛绒方案的问题
3. 评估硅胶方案的优劣
4. 提供详细的方案对比
5. 询问你的确认
6. 更新相关文档和决策记录

### 场景2：提出新功能

**你说：**
```
我想增加一个睡眠监测功能
```

**AI 会做：**
1. 检查当前硬件能力
2. 评估技术可行性
3. 考虑隐私合规性
4. 分析用户价值
5. 提供多个实现方案
6. 与你讨论优先级

### 场景3：调整商业策略

**你说：**
```
考虑首月免费试用来吸引用户
```

**AI 会做：**
1. 读取当前定价策略
2. 分析对收入的影响
3. 参考竞品策略
4. 评估转化率提升
5. 提供建议方案
6. 更新定价文档

## 四、AI 助手的工作流程

当你提出设计想法时，AI 会按以下流程工作：

```
1. 识别话题类型
   └─ 硬件 / 功能 / 商业 / UI 设计

2. 读取背景信息
   ├─ AGENTS.md（项目概述、已有决策）
   ├─ 相关领域文档（硬件规格、产品规格等）
   └─ 检查约束条件（成本、技术、安全）

3. 分析和讨论
   ├─ 理解你的需求和痛点
   ├─ 提供多个方案对比
   ├─ 分析优劣势和成本
   └─ 考虑技术可行性

4. 确认方案
   ├─ 总结讨论结果
   ├─ 明确最终方案
   └─ 获得你的确认

5. 更新文档
   ├─ 更新相关规格文档
   ├─ 添加决策记录到 AGENTS.md
   └─ 更新"最近一次更新"章节
```

## 五、常见问题

### Q1: AI 会自动更新文档吗？

**A:** 是的。根据 AGENTS.md 的自动同步规则，AI 会在确认方案后直接更新相关文档，不会反复询问。更新完成后会告诉你更新了哪些文件。

### Q2: 我可以只讨论不更新文档吗？

**A:** 可以。如果你只是想探讨想法，可以明确说明"我只是想讨论一下，暂时不更新文档"。

### Q3: 如果我不满意 AI 的方案怎么办？

**A:** 你可以：
- 说"我有不同的想法"或"我觉得..."
- AI 会继续与你讨论
- 提供更多方案或调整现有方案
- 直到达成共识

### Q4: AI 会考虑成本和技术可行性吗？

**A:** 会的。AI 在提供方案时会考虑：
- 硬件成本目标（<300元）
- 技术实现难度
- 儿童产品安全标准
- 品牌定位一致性
- 用户体验影响

### Q5: 哪些类型的设计话题可以讨论？

**A:** 任何产品相关的话题都可以，包括：
- 硬件外观、材料、功能
- APP功能、界面设计
- 用户体验改进
- 定价策略调整
- 营销方案优化
- 隐私安全考虑

## 六、最佳实践

### ✅ 推荐做法

1. **描述清楚痛点**
   - 好："毛绒玩具容易脏，小朋友使用难打理"
   - 差："我觉得不好"

2. **说明使用场景**
   - 好："我家孩子喜欢兔子，另一个喜欢海豚"
   - 差："换个造型"

3. **接受 AI 的方案对比**
   - AI 会提供多个方案，耐心看完再决定

4. **明确表达你的决定**
   - 好："确认采用方案A"
   - 差："嗯"、"好"

### ❌ 避免的做法

1. 不要只说"改进一下"而不说具体想法
2. 不要忽略 AI 提供的成本和技术分析
3. 不要在不确定的情况下确认方案
4. 不要期望 AI 猜测你的具体需求

## 七、技术说明

### 背后的工作原理

这个产品设计协作功能基于：

1. **智能文档索引**
   - AI 知道项目中所有文档的位置和用途
   - 根据话题自动读取相关文档

2. **上下文理解**
   - AI 会读取 AGENTS.md 了解项目全貌
   - 理解已有的关键决策，避免重复讨论

3. **自动同步规则**
   - 预定义的文档更新规则
   - 确保所有相关文档保持一致

4. **结构化模板**
   - 针对不同话题类型的讨论模板
   - 确保讨论的完整性和专业性

### 支持文档

- `docs/00-planning/product-design-brainstorm.md` - AI 助手的详细指南
- `AGENTS.md` - 项目上下文和自动同步规则
- `CLAUDE.md` - 项目级别的 AI 指令
