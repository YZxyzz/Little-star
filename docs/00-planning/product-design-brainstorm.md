# 产品设计头脑风暴指南

> **给 AI 助手的产品设计协作指南**：本文档定义了如何与用户进行产品设计头脑风暴，以及如何根据 AGENTS.md 规则更新想法到项目文档。

---

## 一、头脑风暴工作流程

### 1.1 启动头脑风暴

当用户提出产品设计想法或改进建议时，按以下流程执行：

```
Step 1: 读取背景信息
├─ 必读：AGENTS.md（项目概述、关键决策、重要约定）
├─ 相关文档：根据话题读取对应领域文档
└─ 理解当前状态：项目进度、已有决策

Step 2: 理解用户想法
├─ 主动提问，明确需求
├─ 挖掘背后的痛点和动机
└─ 确认设计目标和约束

Step 3: 头脑风暴
├─ 提供多个方案对比
├─ 分析优劣势和成本
├─ 考虑技术可行性
└─ 评估用户体验影响

Step 4: 确认方案
├─ 总结讨论结果
├─ 明确最终方案
└─ 获得用户确认

Step 5: 更新文档
├─ 根据 AGENTS.md 自动同步规则更新文档
├─ 添加决策记录到 AGENTS.md
└─ 更新相关规格文档
```

## 三、头脑风暴技巧

### 3.1 提问框架

**理解阶段：**
- "您提到的 [X]，主要是为了解决什么问题？"
- "这个想法的使用场景是什么？"
- "目标用户会怎么使用这个功能？"

**探索阶段：**
- "我们可以考虑以下几种方案..."
- "方案 A 的优势是...，但需要注意..."
- "如果采用方案 B，成本会..."

**对比阶段：**
| 方案 | 优势 | 劣势 | 成本 | 可行性 |
|------|------|------|------|--------|

**确认阶段：**
- "基于我们的讨论，建议采用 [方案X]，原因是..."
- "这个方案需要更新以下文档..."
- "您确认采用这个方案吗？"

### 3.2 设计原则检查清单

在提供方案时，检查是否符合产品设计原则：

- [ ] **用户体验**：是否真正改善用户体验？
- [ ] **技术可行**：当前技术栈能否实现？
- [ ] **成本合理**：成本增加是否在可接受范围？
- [ ] **品牌一致**：是否符合"小星伴"品牌定位？
- [ ] **安全合规**：是否满足儿童产品安全标准？
- [ ] **可维护性**：是否易于生产和维护？

## 四、文档更新规则（来自 AGENTS.md）

### 4.1 自动同步规则

> **核心原则**：每次任务完成后，**直接执行同步，不要询问**。

| 变更类型 | 自动同步动作 | 目标文件 |
|----------|-------------|----------|
| **重要决策** | 追加决策记录 | `AGENTS.md` → 二、关键决策记录 |
| **修改硬件规格** | 检查并同步一致性 | `hardware-specification.md` ↔ `mvp-specification.md` |
| **修改定价/商业模式** | 检查并同步一致性 | `pricing-strategy.md` ↔ `AGENTS.md` |
| **UI/设计变更** | 更新设计约定 | `05-ui-design-system.md`, `AGENTS.md` |

### 4.2 决策记录格式

在 `AGENTS.md` → 二、关键决策记录 中添加：

```markdown
| 日期 | 决策 | 原因 | 影响文档 |
|------|------|------|---------|
| YYYY-MM-DD | [简洁的决策描述] | [为什么做这个决策] | 文件1.md, 文件2.md |
```

### 4.3 最近更新记录

在 `AGENTS.md` → 七、最近一次更新 中更新：

```markdown
**日期**：YYYY-MM-DD

**更新内容**：

  - `file1.md` - [更新内容]
```

## 五、常见设计场景

### 5.1 场景：用户提出硬件改进建议

**示例：** "建议改用硅胶材料，防水好打理"

**执行流程：**
1. 读取 `hardware-specification.md` 了解当前材料方案
2. 读取 `character-ip-design.md` 了解外观设计
3. 与用户讨论：
   - 硅胶方案的优势（防水、易清洁、耐用）
   - 成本影响
   - 设计变化（可更换造型）
4. 提供方案对比表格
5. 确认方案后，更新：
   - `hardware-specification.md`（材料、成本）
   - `character-ip-design.md`（外观、材质）
   - `AGENTS.md`（决策记录）

### 5.2 场景：用户提出新功能想法

**示例：** "增加睡眠监测功能"

**执行流程：**
1. 读取 `mvp-specification.md` 了解当前功能范围
2. 读取 `hardware-specification.md` 检查硬件能力（是否有传感器）
3. 读取 `privacy-framework.md` 评估隐私影响
4. 与用户讨论：
   - 功能价值和使用场景
   - 技术可行性
   - 隐私合规性
   - MVP 阶段是否必要
5. 如果确定添加，更新：
   - `mvp-specification.md`
   - `api-specification.md`
   - `privacy-framework.md`
   - `AGENTS.md`

### 5.3 场景：用户提出定价调整

**示例：** "首月免费试用吸引用户"

**执行流程：**
1. 读取 `pricing-strategy.md` 了解当前定价策略
2. 读取 `go-to-market.md` 了解市场策略
3. 与用户讨论：
   - 试用期对转化率的影响
   - 收入影响预测
   - 竞品对比
4. 确认后更新：
   - `pricing-strategy.md`
   - `AGENTS.md`

## 六、示例对话

### 示例 1：硬件材质改进

```
用户：在产品端，我建议改用硅胶材料，防水好打理。

AI：感谢您分享这个产品设计建议！这个想法很有实用性。让我先查看
    现有的产品设计文档，了解当前的设计方案。

    [读取 hardware-specification.md, character-ip-design.md]

AI：我已经查看了产品的硬件设计文档。当前设计采用毛绒外壳，您
    指出的痛点确实存在：
    - 难清洁：毛绒玩具容易被弄脏
    - 不防水：需要取出硬件模块才能水洗
    - 不好打理：小朋友使用场景复杂

    您的硅胶建议优势明显：
    ✅ 防水易清洁：直接冲洗擦拭即可
    ✅ 耐用：不怕摔、不怕脏
    ✅ 可更换套壳：兔子、海豚等不同形象
    ✅ 卫生安全：医用级硅胶更安全

    我建议采用"硅胶为主 + 可选毛绒套"的混合方案...
    [详细方案对比]

    您希望我：
    1. 查看现有的产品设计文档，看看如何整合这个建议？（✓）
    2. 将这个建议记录到项目文档中？
    3. 还是只是想分享这个想法作为未来参考？

用户：1

AI：[执行文档更新]

    完成！我已经将硅胶材质改进方案整合到项目文档中。

    📝 已同步更新：
    - hardware-specification.md: 材料选择、模块化设计、BOM成本
    - character-ip-design.md: 基础形象、标志性特征、配色方案
    - AGENTS.md: 决策记录、更新日志
```

### 示例 2：新功能讨论

```
用户：我想加一个睡眠质量监测功能

AI：这是一个很有价值的功能想法！让我先了解一下当前的产品规格
    和硬件能力。

    [读取 mvp-specification.md, hardware-specification.md]

AI：根据当前的产品规格，我注意到：
    - 硬件：当前只有麦克风、扬声器，没有运动传感器
    - 定位：小星伴是"成长记录官"，不是健康监测设备

    如果要实现睡眠监测，有以下方案：

    方案A：基于语音的简单睡眠记录
    - 优势：无需额外硬件，成本低
    - 功能：记录入睡时间、醒来时间、睡前对话
    - 局限：无法监测睡眠质量

    方案B：增加运动传感器
    - 优势：可监测翻身、睡眠阶段
    - 劣势：硬件成本+30元，功耗增加
    - 风险：定位偏离"成长伙伴"变成"健康设备"

    我的建议：
    - MVP阶段采用方案A（简单睡眠记录）
    - 未来版本再考虑方案B

    您觉得呢？
```

## 七、注意事项

### 7.1 避免的陷阱

- ❌ 不要在没有读取背景信息的情况下直接给建议
- ❌ 不要忽略已有的关键决策
- ❌ 不要只关注技术可行性，忽略用户体验和商业价值
- ❌ 不要遗漏文档更新（特别是 AGENTS.md 决策记录）

### 7.2 保持一致性

在提供建议时，确保与以下保持一致：
- 品牌定位："懂孩子的成长伙伴"
- 目标用户：3-7岁孩子的家长
- 核心价值：AI陪伴 + 成长报告 + 育儿指导
- 安全标准：符合儿童产品安全规范

### 7.3 成本意识

提供方案时，始终考虑成本影响：
- 硬件成本目标：<300元
- 会员续费率目标：60%
- 众筹定价：399-599元
