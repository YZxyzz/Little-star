# 用户调研框架

## 一、调研目标

### 1.1 核心问题

1. 验证目标用户画像的准确性
2. 深度理解用户痛点和需求优先级
3. 测试产品概念的接受度
4. 收集定价敏感度信息
5. 了解竞品使用体验和不满

### 1.2 调研对象

| 用户群体 | 样本数 | 筛选条件 |
|---------|-------|---------|
| 核心目标用户 | 30人 | 3-7岁孩子家长，一二线城市，年收入20万+ |
| 边缘用户 | 10人 | 2-3岁或7-8岁孩子家长 |
| 竞品用户 | 10人 | 使用过听力熊/爱小伴等产品的家长 |
| 专家用户 | 5人 | 幼儿教育工作者、儿童心理咨询师 |

---

## 二、调研方法

### 2.1 深度访谈（1对1）

**适用场景**：核心用户、竞品用户、专家用户

**时长**：45-60分钟

**形式**：线上视频/线下面谈

### 2.2 焦点小组（6-8人）

**适用场景**：产品概念测试、定价测试

**时长**：90分钟

**形式**：线上/线下小组讨论

### 2.3 问卷调查

**适用场景**：量化验证访谈发现

**样本量**：500+

**形式**：在线问卷

---

## 三、深度访谈大纲

### 3.1 开场（5分钟）

```
1. 自我介绍和研究目的说明
2. 录音/录像许可获取
3. 强调无标准答案，希望听到真实想法
```

### 3.2 背景了解（10分钟）

```
1. 家庭基本情况
   - 孩子年龄、性别
   - 家庭结构（几个孩子、是否有老人帮带）
   - 父母职业、工作时间
   - 居住城市

2. 日常陪伴情况
   - 平均每天陪伴孩子多长时间？
   - 主要是谁陪伴孩子？
   - 孩子白天主要在哪里？（幼儿园/家/托班）

3. 教育理念
   - 您认为3-7岁最重要的教育是什么？
   - 您对孩子目前的成长满意吗？有什么担忧？
```

### 3.3 痛点挖掘（15分钟）

```
1. 信息断层问题
   - 您每天下班回家后，如何了解孩子这一天的情况？
   - 孩子会主动分享今天的经历吗？
   - 您觉得自己对孩子一天的了解程度如何？（1-10分）
   - 【追问】有没有哪次事后才知道孩子发生了重要的事？

2. 沟通困难
   - 问孩子"今天怎么样"，通常得到什么回答？
   - 您觉得和孩子沟通最大的挑战是什么？
   - 有没有感觉孩子有心事但说不出来的时候？

3. 育儿困惑
   - 最近遇到的最让您困扰的育儿问题是什么？
   - 遇到问题时，您通常怎么寻找答案？
   - 您觉得网上的育儿建议对您有帮助吗？为什么？

4. 焦虑来源
   - 在育儿方面，什么事情最让您焦虑？
   - 您觉得自己是不是有时候会过度干涉孩子？
   - 如果有一个工具能帮您更了解孩子，您最想了解什么？
```

### 3.4 现有解决方案（10分钟）

```
1. 智能产品使用情况
   - 家里有给孩子用的智能产品吗？（智能手表、故事机、学习机等）
   - 【如果有】使用体验如何？最满意和最不满意的地方？
   - 【如果没有】为什么没有购买？有什么顾虑？

2. 录音/记录习惯
   - 您有记录孩子成长的习惯吗？（照片、视频、日记）
   - 如果有人帮您自动记录孩子每天的成长，您觉得有价值吗？
   - 对于录音记录孩子，您有什么顾虑？

3. 竞品评价（针对竞品用户）
   - 为什么选择这款产品？
   - 使用多长时间了？使用频率？
   - 最有价值的功能是什么？
   - 最希望改进的是什么？
   - 续费意愿如何？
```

### 3.5 产品概念测试（15分钟）

```
【展示产品概念】

"小星伴是一款专为3-7岁孩子设计的AI成长伙伴。
它是一个可爱的小挂坠，孩子可以随身佩戴。

对孩子来说：
- 随时可以和小星伴对话，问问题、讲故事
- 不开心的时候可以倾诉，得到安慰和引导

对父母来说：
- 每天收到孩子的成长日报，了解今天发生了什么
- 知道孩子关心什么话题、问了什么问题
- 获得针对性的育儿建议，比如"孩子今天和小朋友起了冲突，建议这样和他聊..."
- 查看长期成长档案，看到孩子的变化轨迹"

1. 第一印象
   - 听完这个介绍，您的第一反应是什么？
   - 哪个功能最吸引您？为什么？
   - 哪个功能您觉得没必要？

2. 使用意愿
   - 如果有这样一个产品，您会考虑购买吗？（1-10分）
   - 什么情况下您会购买？什么情况下不会？

3. 担忧和顾虑
   - 对于这个产品，您最大的担忧是什么？
   - 关于录音和数据，您有什么顾虑？
   - 您担心孩子会过度依赖AI吗？

4. 功能优先级
   - 如果这些功能只能选三个，您会选哪三个？
   - 还有什么功能是您希望有的？

5. 定价敏感度
   - 如果这个产品定价X元（硬件）+ Y元/年（服务），您觉得如何？
   - 您愿意为这样的产品支付多少钱？
```

### 3.6 收尾（5分钟）

```
1. 还有什么我们没聊到但您想分享的？
2. 您愿意参与后续的产品测试吗？
3. 感谢参与，赠送小礼品
```

## 四、焦点小组大纲

### 4.1 产品概念测试组

**参与者**：6-8位核心目标用户

**流程**：
1. 暖场介绍（10分钟）
2. 育儿痛点讨论（20分钟）
3. 产品概念展示（10分钟）
4. 分组讨论（30分钟）
5. 功能优先级排序（15分钟）
6. 总结收尾（5分钟）

### 4.2 定价测试组

**参与者**：6-8位核心目标用户

**测试方法**：Van Westendorp价格敏感度测试

```
1. 在什么价格下，您会觉得这个产品太便宜，质量可能有问题？
2. 在什么价格下，您会觉得这个产品很划算？
3. 在什么价格下，您会觉得这个产品有点贵，但还是可以接受？
4. 在什么价格下，您会觉得这个产品太贵，不会购买？
```

## 六、调研执行计划

### 6.1 时间安排

| 阶段 | 任务 | 时间 |
|------|------|------|
| 准备期 | 招募用户、准备物料 | 第1周 |
| 访谈期 | 完成55场深度访谈 | 第2-4周 |
| 焦点组 | 完成2场焦点小组 | 第3周 |
| 问卷期 | 发放回收问卷 | 第3-4周 |
| 分析期 | 数据分析、报告撰写 | 第5周 |

### 6.2 用户招募渠道

1. 妈妈社群（小红书、微信群）
2. 幼儿园合作
3. 付费招募平台
4. 现有用户推荐

### 6.3 激励机制

| 调研类型 | 时长 | 激励 |
|---------|------|------|
| 深度访谈 | 60分钟 | 200元红包 |
| 焦点小组 | 90分钟 | 300元红包 |
| 问卷调查 | 10分钟 | 抽奖（10%中奖率，50元） |

---

## 七、调研输出物

1. 《用户调研报告》：包含用户画像验证、痛点排序、需求优先级
2. 《竞品用户反馈汇总》：现有产品的优缺点
3. 《产品概念测试报告》：接受度、担忧、建议
4. 《定价研究报告》：价格敏感度分析、推荐定价区间
5. 《用户语录库》：可用于营销的真实用户声音
