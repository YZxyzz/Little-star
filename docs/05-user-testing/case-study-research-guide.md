# 家庭案例研究调研指南

> **给 AI 助手的用户调研协作指南**：本文档定义了如何进行系统化的家庭 case study，以及如何将洞察转化为产品改进建议。

---

## 一、调研工作流程

### 1.1 完整调研流程

```
Step 1: 定义研究目标
├─ 明确本次调研要解决的问题
├─ 确定调研的重点领域（产品/功能/定价/体验）
└─ 设定成功指标

Step 2: 设计用户画像
├─ 基于目标用户（3-7岁孩子家长）
├─ 细分不同家庭类型
└─ 设计代表性案例

Step 3: 设计调研问题
├─ 背景信息收集
├─ 痛点挖掘
├─ 需求验证
└─ 产品概念测试

Step 4: 执行案例研究
├─ 创建虚拟案例或
├─ 记录真实用户访谈
└─ 收集多维度信息

Step 5: 分析和洞察
├─ 提取关键发现
├─ 识别共性痛点
├─ 发现差异化需求
└─ 评估产品匹配度

Step 6: 转化为行动
├─ 产品改进建议
├─ 功能优先级调整
├─ 定价策略优化
└─ 更新项目文档
```

## 二、家庭类型分类框架

### 2.1 核心分类维度

基于小星伴目标用户（3-7岁孩子家长），按以下维度分类：

| 维度 | 分类选项 | 说明 |
|------|---------|------|
| **城市等级** | 一线/新一线/二线 | 影响消费能力和育儿理念 |
| **家庭结构** | 核心家庭/三代同堂/单亲 | 影响育儿参与度和决策 |
| **父母职业** | 双职工/全职妈妈/创业者 | 影响陪伴时间和痛点 |
| **孩子数量** | 独生/二孩/三孩 | 影响预算和需求优先级 |
| **育儿理念** | 科学育儿/传统育儿/自由放养 | 影响产品接受度 |
| **技术接受度** | 科技爱好者/普通用户/技术保守 | 影响产品复杂度容忍度 |

### 2.2 典型家庭画像模板

#### 画像 A：一线城市双职工家庭

```markdown
## 家庭画像：一线城市双职工家庭

### 基本信息
- **城市**：北京/上海/深圳
- **家庭结构**：核心家庭（父母+1-2孩子）
- **父母职业**：双职工，互联网/金融等行业
- **家庭收入**：年收入 50-100万
- **居住情况**：市区小三居

### 家庭成员
- **父亲**：35岁，产品经理，工作繁忙，晚上9点到家
- **母亲**：33岁，设计师，弹性工作，下午6点接孩子
- **孩子**：5岁女儿，幼儿园大班，活泼好奇

### 育儿现状
- **陪伴时间**：工作日每天1-2小时，周末全天
- **育儿参与**：母亲为主，父亲周末参与
- **育儿焦虑**：担心陪伴不够，错过成长关键期
- **现有工具**：早教APP、故事机、绘本

### 痛点与需求
1. **时间焦虑**："工作忙，陪孩子时间少，不知道她白天发生了什么"
2. **沟通困难**："问她幼儿园怎么样，总说'不知道'、'忘了'"
3. **育儿困惑**："不确定自己的教育方式是否正确"
4. **成长记录**："想记录她的成长，但总是没时间整理"
5. **隔代矛盾**："老人带孩子理念不同，但我们不在场"

### 对小星伴的态度
- **价格敏感度**：低（愿意为品质付费）
- **隐私关注度**：高（关注数据安全）
- **技术接受度**：高（科技产品重度用户）
- **核心期待**：
  - 每天了解孩子的真实状态
  - 获得专业的育儿建议
  - 记录成长的珍贵瞬间
```

  - 发现孩子真正的兴趣

## 三、调研问题设计框架

### 3.1 问题设计原则

1. **开放式为主**：避免引导性问题，让用户自由表达
2. **具体场景化**：询问具体场景和例子，而非抽象感受
3. **挖掘动机**：用"为什么"追问，挖掘深层需求
4. **情感共鸣**：关注情绪和感受，不只是功能需求

### 3.2 标准调研问题库

#### 阶段1：背景信息（5-8分钟）

```
家庭结构类：
1. 可以介绍一下您的家庭成员吗？
2. 平时主要是谁在照顾孩子？
3. 您和爱人的工作情况是怎样的？

孩子情况类：
4. 孩子现在多大？上幼儿园了吗？
5. 可以用3个词形容一下孩子的性格吗？
6. 孩子平时最喜欢做什么？

日常生活类：
7. 工作日一天的时间安排大概是怎样的？
8. 每天有多少时间陪伴孩子？
9. 周末通常会做些什么？
```

#### 阶段2：痛点挖掘（10-15分钟）

```
陪伴相关：
10. 您觉得现在陪伴孩子的时间够吗？为什么？
11. 有没有因为工作忙而错过孩子重要时刻的经历？
12. （如果有）当时什么感受？

了解孩子：
13. 您觉得自己了解孩子吗？
14. 有没有哪些时候，您不知道孩子在想什么？
15. 孩子在幼儿园/家里发生的事，您是怎么知道的？

育儿困惑：
16. 育儿过程中，有哪些让您困惑或焦虑的事？
17. 遇到育儿问题，您通常怎么解决？
18. 有没有想过"我这样做对不对"的时候？

成长记录：
19. 您有记录孩子成长的习惯吗？
20. （如果有）是怎么记录的？多久记一次？
21. （如果没有）是什么原因没有记录？
```

#### 阶段3：现有解决方案（5-10分钟）

```
产品使用：
22. 现在用过哪些儿童产品？（早教APP、故事机等）
23. 这些产品解决了什么问题？
24. 有哪些不满意的地方？

信息获取：
25. 育儿知识从哪里获得？（书籍、APP、专家）
26. 您相信这些信息吗？为什么？
27. 有没有信息太多反而不知道怎么办的时候？
```

#### 阶段4：产品概念测试（10-15分钟）

```
概念介绍：
[展示小星伴产品介绍]

28. 您对这个产品的第一印象是什么？

价值感知：
29. 您觉得这个产品能解决您的哪些问题？
30. 最吸引您的功能是什么？为什么？
31. 有哪些功能您觉得没什么用？

顾虑与担心：
32. 听到这个产品，您有什么顾虑吗？
33. 关于隐私安全，您怎么看？
34. 孩子会愿意戴吗？为什么？

购买决策：
35. 如果这个产品定价399元，您会考虑购买吗？
36. （如果会）什么因素会让您下单？
37. （如果不会）什么因素阻碍了您？
38. 您觉得合理的价格是多少？为什么？

使用场景：
39. 如果买了，您觉得孩子什么时候会戴？
40. 有哪些场景您不希望孩子戴？
41. 您会让孩子每天都戴吗？

期待与建议：
42. 如果可以增加一个功能，您希望是什么？
43. 有没有什么担心这个产品做不到的事？
44. 对这个产品还有什么建议？
```

#### 阶段5：深度挖掘（弹性）

```
情感层面：
- 如果小星伴真的能让您更了解孩子，对您意味着什么？
- 您希望孩子长大后记得什么？

竞品对比：
- 您听说过类似的产品吗？（Mooni M1、听力熊等）
- 和它们相比，您觉得小星伴有什么不同？

传播意愿：
- 如果您买了并且满意，会推荐给朋友吗？
- 您会怎么介绍这个产品？
```

## 五、定价策略建议

### 价格接受度分析

| 价格区间 | 接受度 | 用户群体 |
|---------|-------|---------|
| <300元 | 90% | 全部用户 |
| 300-400元 | 60% | A类、C类 |
| 400-500元 | 30% | C类（高收入） |
| >500元 | 10% | 极少数 |

### 建议定价策略

**方案推荐**：
- 众筹早鸟价：299元（前100名）
- 众筹价：349元
- 正价：399元
- 首年会员：免费（价值358元）

**理由**：
1. 299元早鸟价击穿 Mooni M1（312元）
2. 349元众筹价保持竞争力
3. 399元正价匹配价值感知
4. 首年免费会员提升总价值

## 六、营销话术优化

### 现有话术问题

用户反馈：
- "成长记录官是什么意思？"（6/10 不理解）
- "和其他AI产品有什么不同？"（8/10 疑问）

### 建议话术

**主打信息**：
> "每天10分钟报告，读懂孩子的小心思"

**对比信息**：
> "不是AI对话机，是孩子的成长记录官"

**价值信息**：
> "了解孩子 + 育儿指导，帮你成为更好的父母"

## 六、虚拟案例创建指南

当没有真实用户访谈时，可以创建虚拟案例进行分析：

### 6.1 虚拟案例创建原则

1. **基于真实场景**：参考行业报告、育儿论坛、用户评论
2. **细节丰富**：具体到时间、地点、对话、情绪
3. **痛点真实**：来源于真实育儿问题
4. **多样性**：覆盖不同家庭类型

### 6.2 虚拟案例模板

```markdown
# 虚拟案例：[家庭名称]

## 场景描述

周五晚上8点，北京朝阳区某小区。

张女士（35岁，产品经理）刚到家，5岁的女儿朵朵已经睡了。她打开手机看了看家里的监控回放，看到女儿和奶奶吃晚饭、看动画片的片段。

"又是看动画片..." 张女士叹了口气。她很想知道女儿今天在幼儿园发生了什么，但现在孩子睡了，明天早上问她也记不清了。

## 痛点场景1：沟通困难

**周六早晨**

张女士："朵朵，昨天幼儿园好玩吗？"
朵朵："嗯。"（继续吃饭）
张女士："都做了什么呀？"
朵朵："不记得了。"
张女士："交到新朋友了吗？"
朵朵："不知道。"

张女士感到挫败。她知道女儿在幼儿园一定发生了很多事，但女儿就是不愿意说，或者说不清楚。

## 痛点场景2：时间焦虑

**周一晚上**

张女士加班到9点才到家，女儿已经睡了。这是这周第三次了。

她打开手机相册，翻看女儿的照片，突然意识到：自己已经好几天没有好好和女儿说话了。

"我错过了她的成长..." 这个念头让她难过。

## 痛点场景3：育儿困惑

**周三接到老师电话**

老师："张女士，朵朵最近有点不合群，总是一个人玩。"

张女士很意外，她觉得女儿在家很活泼啊。是自己不了解女儿吗？还是女儿在学校和在家不一样？

她不知道该怎么办，在育儿群里问，得到各种建议，反而更困惑了。

## 对小星伴的反应

**看到产品介绍后**

张女士眼睛亮了："如果能每天知道女儿在想什么，那太好了！"

她最关心的问题：
1. "真的能了解孩子吗？还是只是录音？"
2. "报告会说什么？会给具体建议吗？"
3. "女儿会愿意戴吗？会不会觉得丑？"
4. "隐私安全吗？录音会不会被泄露？"

**价格反应**

"399元...不算贵，如果真有用的话。但能试用一下吗？万一女儿不愿意戴呢？"

## 洞察总结

**核心痛点**：
1. 工作忙，错过孩子成长（时间焦虑）
2. 孩子不说，不知道她的真实状态（沟通困难）
3. 遇到问题，不知道怎么办（育儿困惑）

**核心需求**：
1. 每天了解孩子的状态（即使不在场）
2. 发现孩子的变化（性格、兴趣、情绪）
3. 获得专业的育儿建议（不是泛泛而谈）

**产品匹配度**：
- 每日报告：★★★★★（核心需求）
- 育儿指导：★★★★★（核心需求）
- AI陪伴：★★★☆☆（次要需求）
- 可穿戴：★★★★☆（担心孩子不愿意戴）

**决策因素**：
- ✅ 如果有试用期 → 购买意愿高
- ✅ 如果有真实案例展示 → 信任度高
- ⚠️ 担心孩子不愿意戴 → 需要解决
- ⚠️ 担心隐私 → 需要详细说明
```
