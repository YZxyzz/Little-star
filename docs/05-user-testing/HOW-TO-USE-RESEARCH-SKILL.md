# 如何使用用户调研功能

> 本文档说明如何与 AI 助手进行系统化的用户案例研究

---

## 一、快速开始

### 方式1：直接发起调研

只需说出你的调研需求：

```
"创建一个双职工家庭的案例研究"
"帮我分析二线城市全职妈妈的痛点"
"设计一个调研问卷，测试产品定价"
```

AI 会自动：
1. 读取项目背景（AGENTS.md、产品规格等）
2. 选择或创建用户画像
3. 执行案例研究或设计调研问题
4. 分析洞察并提出改进建议
5. 更新相关项目文档

### 方式2：使用关键词触发

| 关键词 | 触发内容 |
|--------|---------|
| 案例研究/case study | 创建虚拟案例或记录真实访谈 |
| 用户画像/persona | 设计或细化用户画像 |
| 调研问题/访谈问题 | 生成调研问题清单 |
| 痛点分析/需求分析 | 分析用户痛点和需求 |
| 定价测试 | 测试用户价格接受度 |

## 二、典型使用场景

### 场景1：验证产品概念

**你说：**
```
"创建几个案例，测试用户对小星伴的接受度"
```

**AI 会做：**
1. 创建3-5个不同类型的家庭画像
2. 模拟他们看到产品后的反应
3. 记录积极反馈和顾虑
4. 评估产品功能匹配度
5. 提出产品优化建议

**输出示例：**
```markdown
# 案例研究总结

## 画像A：一线城市双职工家庭
核心痛点：时间焦虑，错过成长
对小星伴：★★★★★ 高度匹配
愿付价格：400-500元

## 画像B：二线城市全职妈妈家庭
核心痛点：二孩精力分配
对小星伴：★★★★☆ 较匹配
愿付价格：300-400元

## 共性发现
1. "每日报告"是最受欢迎功能
2. 价格敏感度：低收入家庭关注性价比
3. 主要顾虑：孩子是否愿意戴

## 建议行动
- 调整定价策略（增加早鸟价）
- 强化"每日报告"的营销
- 设计试戴期降低顾虑
```

### 场景3：测试定价策略

**你说：**
```
"测试用户对399元定价的接受度"
```

**AI 会做：**
1. 创建不同收入层级的家庭画像
2. 模拟他们对定价的反应
3. 分析价格敏感度差异
4. 提供定价优化建议

**输出示例：**
```markdown
# 定价接受度分析

## 高收入家庭（年入80万+）
- 价格敏感度：低
- 399元反应："不贵，效果好就行"
- 决策因素：功能 > 价格

## 中收入家庭（年入40-80万）
- 价格敏感度：中
- 399元反应："可以接受，但要看性价比"
- 决策因素：性价比

## 建议
- 主力价格：349元（覆盖中高收入）
- 早鸟价：299元（吸引中等收入）
- 强化"首年免费会员"（价值358元）
```

### 场景4：对比竞品

**你说：**
```
"分析用户对小星伴和 Mooni M1 的态度"
```

**AI 会做：**
1. 让虚拟用户了解两款产品
2. 记录他们的对比反应
3. 分析选择偏好和原因
4. 提出差异化策略

**输出示例：**
```markdown
# 竞品对比用户反应

## 用户A（注重深度）
"Mooni M1 功能很多，但感觉不够专注。

原因：价格敏感

## 策略建议
- 强化"成长记录官"定位
- 价格调整到349元缩小差距
- 营销强调深度理解 vs 功能堆砌
```

## 六、虚拟案例 vs 真实访谈

### 虚拟案例（无真实用户时）

**优势：**
- 快速创建，不需要招募用户
- 可以覆盖多种画像类型
- 成本低，迭代快

**局限：**
- 基于假设，需要后续验证
- 可能遗漏真实痛点

**适用场景：**
- 产品早期，快速验证假设
- 探索不同用户类型的需求
- 设计真实访谈前的准备

### 真实访谈（有真实用户时）

**优势：**
- 真实痛点，可信度高
- 发现意想不到的洞察
- 用户原话有感染力

**局限：**
- 需要招募用户，成本高
- 样本量小，代表性有限

**适用场景：**
- 验证虚拟案例的假设
- 深度挖掘特定痛点
- 准备营销素材（真实故事）

## 八、最佳实践

### ✅ 推荐做法

1. **先虚拟后真实**
   - 用虚拟案例快速探索
   - 再用真实访谈验证

2. **覆盖多种画像**
   - 至少3-5种不同类型家庭
   - 覆盖不同收入、职业、城市

3. **记录用户原话**
   - 用引号标注真实表达
   - 用户原话比总结更有力

4. **持续迭代**
   - 不是一次性调研
   - 产品迭代过程中持续收集

### ❌ 避免的做法

1. ❌ 只做一种类型的案例（样本偏差）
2. ❌ 只关注积极反馈（忽视顾虑）
3. ❌ 做完就放着不用（浪费洞察）
4. ❌ 强行套用结论（脱离实际）

## 九、常见问题

### Q1: 虚拟案例靠谱吗？

**A:** 虚拟案例基于：
- 真实育儿场景
- 行业报告数据
- 用户评论分析
- 常见育儿痛点

**建议：**
- 早期用虚拟案例快速迭代
- 关键决策前用真实访谈验证

### Q2: 需要多少个案例？

**A:** 分阶段：
- 第1轮：5-10个案例（验证核心假设）
- 第2轮：10-20个案例（细化画像）
- 持续收集（跟踪市场变化）

### Q3: 案例研究能代替真实用户测试吗？

**A:** 不能完全代替，但可以：
- 在真实测试前快速验证方向
- 设计更好的测试方案
- 降低试错成本

真实用户测试仍然必不可少。

### Q4: 如何确保案例的多样性？

**A:** 系统化覆盖：
- 城市：一线/新一线/二线
- 收入：高/中/低
- 职业：双职工/全职/创业者
- 家庭：核心/三代/单亲
- 孩子：独生/二孩/三孩
