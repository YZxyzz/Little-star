# 可穿戴设备硬件规格说明

## 一、设计目标

### 1.1 核心设计原则

1. **安全第一**：所有材料符合儿童产品安全标准
2. **轻量便携**：让孩子愿意全天佩戴
3. **持久续航**：满足一整天使用需求
4. **可爱吸引**：孩子主动愿意佩戴
5. **耐用防护**：适应儿童使用场景

### 1.2 目标规格概览

> ⚠️ **重要更新 (2025-02-06)**：产品形态调整为**儿童吊坠/项链**，尺寸缩小至约 5cm，采用 **Smart VAD 智能省电技术** 提升续航。

| 参数 | 目标值 | 说明 |
|------|--------|------|
| 总重量 | **<30g** | 极致轻量化，佩戴无负担 |
| 硬件模块尺寸 | **30×30×10mm** | 高度集成方案 |
| **续航** | **>24小时** | 500-600mAh电池，满足全天候使用 |
| 充电时间 | <2小时 | USB-C快充 |
| **网络** | **内置蜂窝流量卡** | Unet Air780E等超小模组 |
| 防水等级 | 生活防水 | - |
| 工作温度 | -10°C ~ 45°C | - |
| 存储容量 | 16GB | 离线缓存 |
| 振动反馈 | 支持 | 微型扁平马达 |
| **实时传输** | **支持** | 蜂窝网络实时上传音频 |

### 2.2 核心组件选型

#### 主控芯片

| 候选 | 型号 | 特点 | 推荐度 |
|------|------|------|--------|
| **方案A** | **ESP32-S3 + 4G模组** | BLE配对 + 蜂窝数据传输 | **★★★★★** |
| 方案B | nRF9160 | 集成LTE-M/NB-IoT, 低功耗 | ★★★★☆ |
| 方案C | 高通MDM9206 | 全功能LTE Cat.M1 | ★★★☆☆ |

**推荐**：ESP32-S3 + 移远EC20/SIM7600 4G模组
- ESP32-S3负责音频处理、BLE配对
- 4G模组负责蜂窝数据传输（Cat.1/Cat.4）
- 内置eSIM或Nano SIM卡槽

#### 麦克风

| 候选 | 型号 | 特点 | 推荐度 |
|------|------|------|--------|
| 方案A | Knowles SPH0645LM4H | 数字输出, 高信噪比 | ★★★★★ |
| 方案B | InvenSense ICS-43434 | 低功耗, 高性能 | ★★★★☆ |

**推荐**：双麦克风阵列（降噪需要）

#### 存储

| 候选 | 容量 | 特点 | 推荐度 |
|------|------|------|--------|
| 方案A | 8GB NAND Flash | 大容量, 成本适中 | ★★★★★ |
| 方案B | 4GB NAND Flash | 成本更低 | ★★★☆☆ |

**推荐**：8GB（可存储约250小时音频）

#### 电池

> ⚠️ **更新**：为适配吊坠形态，电池容量调整为 600mAh 左右。

| 候选 | 容量 | 特点 | 推荐度 |
|------|------|------|--------|
| **方案A** | **600mAh Li-Po** | **满足24h+续航** | **★★★★★** |
| 方案B | 500mAh Li-Po | 体积更小，续航稍弱 | ★★★★☆ |
| 方案C | 1200mAh Li-Po | 太大太重，不适合吊坠 | ★☆☆☆☆ |

**推荐**：600mAh（约24-28小时续航，每晚充电）

**续航计算**：
- 持续录音功耗：约20mA
- 蜂窝网络间歇传输：平均8mA
- 总平均功耗：约28mA
- 续航时间：600mAh ÷ 28mA ≈ 21小时（实际使用约25小时，因有静默期）

**推荐**：1027扁平马达（适配吊坠体积）

### 2.3 详细模组选型 (Shenzhen Supply Chain)

> 基于华强北供应链的可获取性与成本效益分析。

| 模块分类 | 推荐型号 | 主要参数 | 深圳市场状态 | 参考单价(CNY) |
| :--- | :--- | :--- | :--- | :--- |
| **主控 (MCU)** | **ESP32-S3-WROOM-1** | 双核Xtensa, WiFi+BLE, AI指令集 | **标品** | ~13-15 |
| **4G模组** | **Air780E** (合宙) | LTE Cat.1, 极小尺寸, 低功耗 | **爆款** | ~20-25 |
| **麦克风** | **INMP441** | I2S 数字输出, 全向 | **通用** | ~3-5 |
| **功放** | **MAX98357A** | I2S 输入, Class D | **通用** | ~2-5 |
| **电池** | **602530** (600mAh) | 3.7V 锂聚合物 | **易定制** | ~10-12 |

**选型理由**：
1.  **ESP32-S3**: 生态极佳，支持 Arduino/IDF，自带 I2S 接口可直接处理数字音频，省去额外 Codec 成本。
2.  **Air780E**: 目前市面上体积最小的 Cat.1 模组之一，非常适合吊坠形态，且价格极具优势。
3.  **纯数字音频链路**: 麦克风和大功率放均采用 I2S 接口，抗干扰能力强，电路设计简单。

### 3.6 安全设计

1. **无小零件**：眼睛牢固嵌入，所有配件固定连接
2. **安全挂绳**：超过2kg拉力自动断开，防勒颈
3. **磁铁安全**：磁铁完全包裹在硅胶内，防止误吞
4. **圆角设计**：硬件模块所有边缘圆滑处理
5. **阻燃材料**：毛绒外壳符合阻燃标准
6. **可水洗设计**：取出硬件模块后，毛绒外壳可机洗
7. **材料安全**：
   - 符合GB 6675玩具安全标准
   - 符合EN 71欧盟玩具安全标准
   - 无邻苯二甲酸酯
   - 无双酚A
   - 无重金属

#### 智能录音模式 (Smart VAD)

> **核心机制**：为了极大省电，设备不会24小时全功率录音，而是采用 **VAD (语音活动检测)** 技术。

1.  **极低功耗监听**：平时设备处于"半休眠"状态，只用极少电量监听"有没有人说话"。
    *   *环境音/噪音*：忽略，不唤醒。
    *   *安静时*：停止采集，进入休眠。
2.  **自动唤醒 (Auto-Wakeup)**：
    *   检测到**人声频率**：设备毫秒级唤醒，开始高保真录音并缓存。
    *   *进阶功能 (可选)*：**声纹辅助过滤**。在云端或本地简单识别是否为"主人"的声音，如果不是则丢弃数据，进一步减少无效上传。
3.  **按键干预**：
    *   任何时候按下按钮，强制唤醒并开始对话。

**功耗优势**：
通过 VAD 技术，在没有说话的安静场景（如午休、上课发呆、夜间），电量消耗仅为正常工作的 1/10。这是实现小电池长续航的关键。

| 参数 | 规格 |
|------|------|
| 马达类型 | ERM 或 LRA |
| 振动强度 | 可调节（轻/中/强）|
| 响应时间 | <20ms |
| 功耗 | <50mA（工作时）|

3.  **主控芯片 (MCU)**: 产品的"大脑" (ESP32)，负责指挥。
4.  **功放芯片 (Amp)**: 产品的"肌肉"，放大声音推动喇叭。
5.  **Smart VAD**: "智能静默监听"，像门卫一样，只有听到人说话才唤醒系统，极大省电。
