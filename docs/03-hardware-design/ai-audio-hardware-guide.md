## 一、四款主流产品硬件模块全拆解

所有 AI 录音硬件的核心都可以拆成 **6 大模块**。下面逐一讲解每个模块是什么、干什么用，以及各家用了什么芯片/方案。

### 1.1 六大核心硬件模块及其作用

| 模块 | 通俗解释 | 作用 |
|------|----------|------|
| **主控芯片（SoC）** | 相当于"大脑" | 负责控制整个设备的运行，管理录音、存储、通信等所有功能 |
| **麦克风（MIC）** | 相当于"耳朵" | 采集声音。MEMS 硅麦是微型数字麦克风，体积极小音质好；骨传导麦克风通过震动采集声音，贴手机时录通话特别清晰 |
| **存储芯片** | 相当于"硬盘" | 存放录音文件。64GB 大约能存 400+ 小时录音 |
| **无线通信模块** | 相当于"嘴巴"（对外传输） | 蓝牙用于连接手机 App、Wi-Fi 用于快速传输大文件到手机或云端 |
| **电池 + 电源管理** | 相当于"心脏" | 提供电力，决定续航时长。电源管理芯片负责充放电安全 |
| **人机交互** | 相当于"脸面" | 按键、指示灯、振动马达、OLED 屏等，让用户知道设备在干什么 |

## 三、硬件产品从图纸到实物的全流程术语详解

如果你是硬件外行，下面这些术语是你必须搞懂的。整个流程就像"盖房子"：

**设计图纸 → 做电路板 → 把零件焊上去 → 做外壳 → 组装成品**

### 3.1 ID 设计师 = 外观设计师

**ID = Industrial Design（工业设计）**

决定你的产品**长什么样子**的人。比如 PLAUD 那个卡片的圆角弧度、按键位置、Logo 放哪里、是银色还是黑色——全是 ID 设计师决定的。他们交付：产品效果渲染图、三维模型文件。

### 3.2 结构设计师 = 内部空间规划师

ID 设计师决定了外观之后，结构设计师要解决**里面怎么塞得下**的问题：电池放哪里、电路板怎么固定、麦克风开孔位置、外壳上下两半怎么扣合、按键怎么联动到电路板上。

**简单说：ID 设计师画"皮"，结构设计师设计"骨架"。**

> 💡 好的 ID 设计师确实要懂结构，设计才合理。大公司会拆成两个岗位（各自深度要求不同，需要反复沟通妥协），但小团队/个人创业者通常一个人兼顾。对于录音卡片这种相对简单的产品，一个人同时做 ID + 结构完全可行。

### 3.3 PCB = 电路板（那块绿色/蓝色的硬板子）

**PCB = Printed Circuit Board（印刷电路板）**

拆开任何电子产品看到的那块硬板子，上面有密密麻麻的铜线路。它的作用是**把所有芯片和零件连接起来**，相当于城市的道路网络，让电信号从 A 芯片流到 B 芯片。

PCB 生产就是把你的电路设计图变成实物板子的过程。

**打个比方：PCB 是一块空白的城市地图——只有路，没有建筑。**

### 3.4 SMT 贴片 = 把零件焊到电路板上

**SMT = Surface Mount Technology（表面贴装技术）**

PCB 做好之后是空的，上面没有任何芯片和零件。SMT 就是用机器把芯片、电阻、电容这些微小零件**精确地放到 PCB 上并焊接牢固**的过程。

流程：先在 PCB 上刷一层锡膏（像涂胶水）→ 机器高速把零件一个个放上去 → 过回流焊炉（高温让锡膏熔化把零件焊死）。

**打个比方：SMT 就是在空白地图上盖好所有建筑。**

### 3.5 PCBA = PCB + 焊好零件的成品板

**PCBA = PCB Assembly（电路板组装）**

就是"空板子 + 焊好所有零件"的总称。"PCBA 工厂"意味着这家工厂能**一条龙帮你做板子 + 焊零件**，交到你手里就是可以通电工作的电路板了。

### 3.6 CNC = 金属外壳加工方式

**CNC = Computer Numerical Control（数控机床）**

想象一块铝块，CNC 就是用电脑控制的刀具，一刀一刀把它**削成**你要的形状。PLAUD 那个 3mm 超薄铝合金外壳就是这样做出来的。

CNC 的特点：
- **不需要开模具**（省了模具钱），直接从金属块上切
- 精度极高，质感好
- 但**单件成本贵**（每一件都要慢慢削），适合金属外壳
- 量大了也还行，苹果 MacBook 外壳也是 CNC

与之对比的是**注塑**（塑料外壳）：先花几万块做一个模具（像蛋糕模子），然后往里面灌塑料，一秒出一个，单件超便宜但模具贵。

### 3.7 完整流程串联图

```
你的想法
  ↓
ID设计师 → 画出产品外观（好看不好看）
  ↓
结构设计师 → 设计内部空间（塞不塞得下）
  ↓
硬件工程师 → 设计电路原理图和PCB布局
  ↓
PCB工厂 → 生产空白电路板
  ↓
SMT工厂 → 把芯片零件焊到板子上 → 得到 PCBA
  ↓
CNC工厂 / 注塑工厂 → 生产外壳
  ↓
组装厂 → 把 PCBA + 电池 + 麦克风 + 外壳组装 → 成品
```

> 💡 实际操作中，找一家深圳的方案公司或 PCBA 工厂，他们通常能帮你搞定从 PCB 生产到 SMT 贴片的全部流程，你不需要分别找不同工厂。嘉立创（立创商城旗下）就是国内最知名的一站式 PCB+SMT 平台，小批量打样非常方便。

## 四、开发板是什么？——理解硬件开发的起点

### 4.1 开发板的概念

**开发板 = 芯片厂商帮你把芯片焊好在一块板子上，接好了电源、USB 口、引脚接口，让你直接拿来写程序、接传感器做实验用的"半成品板子"。**

打个比方：芯片就像一颗裸露的发动机，你没法直接用——需要接油管、接电线、接排气管才能转。开发板就是厂商帮你把这些全接好，你插上电就能转的"发动机试验台"。

你在开发板上验证功能、跑通原型之后，再请硬件工程师把电路浓缩到你自己的小 PCB 上，这就是从"开发板原型"到"产品级硬件"的过程。

### 4.2 常见开发板对比

| 开发板 | 芯片 | 特点 | 价格 | 适合做录音硬件吗 |
|--------|------|------|------|-----------------|
| **ESP32-S3** | 乐鑫 ESP32-S3 | 自带 Wi-Fi + 蓝牙 + 音频接口(I2S)，生态极好，教程最多 | ¥20-50 | ✅ **最推荐，天然适合** |
| **STM32 核心板** | ST 意法半导体 STM32 系列 | 工业级单片机，稳定可靠，资料极多，全球用量最大 | ¥15-60 | ⚠️ 能做录音，但本身不带 Wi-Fi/蓝牙，需额外加模块 |
| **树莓派 Zero W** | 博通 BCM2835 | 跑 Linux 系统的微型电脑，自带 Wi-Fi | ¥80-120 | ⚠️ 功耗大、体积大，不适合便携产品，适合桌面原型 |
| **Nordic nRF52840** | Nordic nRF52840 | 超低功耗蓝牙，PLAUD 就用了 Nordic 的芯片 | ¥40-80 | ✅ 适合做最终产品，但开发门槛高一些 |

### 4.3 STM32 到底是什么

STM32 是意法半导体（ST）公司生产的一个系列**单片机**芯片。"单片机"就是一颗包含了处理器、内存、各种接口的芯片，可以独立运行程序。STM32 是全球用量最大的单片机之一，大量工业产品、家电、医疗设备都用它。

它和 ESP32 的核心区别：**STM32 本身不带 Wi-Fi 和蓝牙**（部分高端型号除外），更偏向"纯控制"；**ESP32 自带无线通信**，更适合需要联网的产品。对于录音硬件来说，**ESP32-S3 是更好的起步选择**。

## 八、能不能自己做？——全流程自学能力地图

### 8.1 全流程所需能力拆解

| 环节 | 需要学的技能 | 自学难度 | 建议 |
|------|-------------|---------|------|
| **产品定义** | 想清楚做什么、给谁用、差异点 | ★☆☆☆☆ | 你来做，这是创始人的核心工作 |
| **外观设计（ID）** | Fusion 360 / SolidWorks 建模 | ★★★☆☆ | 可以自学，录音卡片形态很简单 |
| **结构设计** | 理解壁厚、卡扣、公差配合 | ★★★☆☆ | 和 ID 一起学，简单产品能搞定 |
| **电路设计** | 画原理图 + PCB 布局（立创 EDA） | ★★★★☆ | 建议外包或用现成开发板替代 |
| **嵌入式编程** | C/Arduino 语言，控制录音+蓝牙+存储 | ★★★★☆ | ESP32 有大量教程，可以硬啃 |
| **App 开发** | Flutter / React Native / 原生 | ★★★★☆ | 建议外包，或先用微信小程序简化 |
| **AI 转写对接** | 调用讯飞/阿里云/OpenAI 的语音转文字 API | ★★☆☆☆ | 不难，就是调接口 |
| **量产对接** | 和工厂沟通、BOM 采购、品控 | ★★★☆☆ | 到了这步再学也来得及 |

### 8.2 推荐自学路线和时间表

**第一个月：建立基础认知**

- **第 1-2 周**：买一块 ESP32-S3 开发板 + INMP441 麦克风模块（总共不到 ¥100），跟 B 站教程实现"录音并保存到 SD 卡"。关键词搜索：`ESP32 I2S 录音 教程`。这一步让你理解嵌入式开发的基本逻辑。
- **第 3-4 周**：在上面加蓝牙功能，让手机能通过蓝牙接收录音文件。搜索：`ESP32 BLE 文件传输`。

**第二个月：学外观和结构**

- 学 **Fusion 360**（免费个人版），建模一个简单的方盒子外壳，包含按键开孔和麦克风孔位。B 站搜 `Fusion 360 入门教程`，一周能上手基本操作。
- 用 3D 打印把外壳打出来（淘宝 3D 打印服务几十块钱），把你的开发板塞进去感受一下。

**第三个月：做出可演示的 MVP**

- 把录音 + 蓝牙传输 + 手机端简单界面串通，做出一个能给别人演示的完整原型。
- 手机端可以先用现成工具（比如蓝牙串口调试 App），不急着开发正式 App。

**第四个月起：决定后续路线**

- 到这个阶段你已经有了足够的认知来做判断——是继续自己深入做电路设计，还是找方案公司帮你把 MVP 变成产品级方案。

### 8.3 关键认知

> **你不需要成为每个环节的专家，但你需要每个环节都懂到"能和专业人对话、能判断他做得对不对"的程度。** 自己做 MVP 原型大约需要 2-3 个月的集中学习，但把它变成可以卖的产品，最终还是要和专业的人合作。你的核心价值在于产品定义和商业判断，不在于亲手焊电路板。

## 九、关键建议与行动计划

### 对于完全外行的你，建议路径是：

1. **第一步（本周）**：花 ¥100-200 买一套 ESP32-S3 + INMP441 麦克风模块，找 B 站教程跑通录音功能，建立感性认知

2. **第二步（1-2 周）**：买一台 PLAUD Note 或 DingTalk A1 实际使用，深入理解用户体验和功能边界

3. **第三步（2-4 周）**：带着你的想法（差异化功能点）去深圳找 2-3 家方案公司聊，获取报价和时间线

4. **第四步**：确认方案公司后启动项目，同步开发手机 App 和 AI 后端

### 最重要的认知：

> **这类产品的核心竞争力不在硬件，而在软件和生态。** 硬件部分是高度成熟的，芯片、麦克风、电池都是现成方案。真正决定成败的是：AI 转写的准确率、App 的使用体验、以及你切入的差异化场景。

出门问问 CEO 李志飞说过一句话很到位：「我们在硬件上面栽了太多的跟头，现在认为重点应该是做好软件。不为了做硬件而做硬件，而是做好软件服务，再用成熟的硬件去实现功能。」
